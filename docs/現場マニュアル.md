# TD判断AI 現場マニュアル

> 大会現場で TD・フロアスタッフが使うためのセットアップと使用手順
> 2026-04-08 初版

---

## 🎯 このツールでできること

- ポーカートーナメント中に起きた判断ケースを入力 → **10-30 秒で AI が判断を返します**
- TDA 2024 年版の全ルールを参照済み
- 日本語で入力 → 日本語で判断が返る
- スマホからでも PC からでも使える
- 判断履歴が全部保存される

**⚠️ 最終判断は必ず現場の人間 TD が行ってください。AI は「推奨」を出すだけです。**

---

## 🚀 初回セットアップ（一度だけ）

### 1. API Key を取得

1. https://console.anthropic.com/settings/keys を開く
2. ログイン → 「Create Key」
3. 名前を付ける（例: `P1-TD-AI`）
4. `sk-ant-...` で始まる文字列をコピー（**これは一度しか表示されません**）

### 2. API Key を設定

Mac のターミナルで：

```bash
cd ~/Desktop/Hidane-AI/AI基盤/TD-AI
cp .env.example .env
open .env  # テキストエディタで開く
```

`ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxxxxxxxxx` の `xxxxx...` の部分を、さっきコピーしたキーに置き換えて保存してください。

### 3. 起動する

```bash
bash start_server.sh
```

初回は必要なライブラリを自動でインストールします（1-2 分）。

以下のようなメッセージが出たら成功です：
```
📍 アクセス URL:
   PC:     http://localhost:8501
   スマホ: http://192.168.xx.xx:8501
```

ブラウザが自動で開きます。

## 📱 スマホからアクセスする方法

**前提**: PC とスマホが**同じ Wi-Fi** に接続されていること

1. 上の起動メッセージの「スマホ:」の URL をスマホのブラウザで開く
2. PC と同じ画面が表示される
3. そのままフォームに入力して使える

**ホーム画面に追加**（Safari）:
1. Safari でアクセス
2. 共有ボタン → 「ホーム画面に追加」
3. アプリのようにワンタップ起動できる

## 🎮 使い方（現場での実際の流れ）

### ステップ 1: 状況を入力

画面中央の大きなテキストエリアに、起きた状況を自然な日本語で書く：

**悪い例**（情報不足）:
```
プレイヤーが文句言ってる
```

**良い例**（判断に必要な情報が揃ってる）:
```
Day 1 後半、NLHE、Blinds 2,000-4,000 BBA 4,000。
UTG が 12,000 オープン、CO が無言で 5,000×2 + 1,000×2 = 12,000 を押し出した。
UTG から「これはコール？レイズ？」の声。
ディーラーが時計を止めてフロアを呼んだ。
```

**必ず入れるべき情報**:
- トーナメントフェーズ（Day1 後半、Final Table など）
- ブラインド
- 誰がどんなアクションをしたか
- 論点（何を判断してほしいか）

### ステップ 2: 追加情報を選ぶ（任意）

- **トーナメントフェーズ**: 「Day 1 後半」「Final Table」など
- **ブラインド**: 「2,000-4,000 BBA 4,000」など
- **ゲーム種目**: NLHE（デフォルト）/ PLO / HORSE など

（空欄でも OK。状況文に含まれていれば自動で読み取ります）

### ステップ 3: 「判断を取得」ボタンを押す

10-30 秒待つと、判断が表示される：

```
【状況の要約】
【適用ルール】主要 + 補助 + ペナルティ
【推奨判断】
【根拠】
【代替案】（あれば）
【ペナルティ】（該当時）
【確認事項】
【確信度】high / medium / low
```

### ステップ 4: フィードバックを残す

判断を見たあと、必ず 3 ボタンのどれかを押してください：

- ✅ **正しい** — AI の判断が完璧
- 🟡 **部分的** — 一部正しいが改善余地あり
- ❌ **間違い** — AI の判断が違っていた

**このフィードバックでシステムが学習します。使うほど賢くなります。**

詳細コメント（任意）:
- 「現場では別の判断をした」
- 「改善点」
- 「入力者名（中野 / TD名）」

## 🔧 トラブルシューティング

### サーバーが起動しない
```bash
cd ~/Desktop/Hidane-AI/AI基盤/TD-AI
# 依存関係を再インストール
pip3 install streamlit anthropic scikit-learn pypdf --upgrade
# 再試行
bash start_server.sh
```

### 「ANTHROPIC_API_KEY が設定されていません」と出る
```bash
# .env ファイルを確認
cat .env
# キーが正しく記載されているか確認
```

### スマホからアクセスできない
- PC とスマホが**同じ Wi-Fi** に接続されているか確認
- PC のファイアウォールが 8501 ポートをブロックしていないか確認
- PC の IP アドレスが変わった場合は、再起動して新しい IP を確認

### 判断が遅い（30 秒以上）
- ネット接続を確認
- API Key のクレジット残高を確認: https://console.anthropic.com/settings/billing

### 判断の精度が悪い
1. 状況文に必要な情報が含まれているか確認（ブラインド、アクション、論点）
2. ❌ フィードバックを残す（次回から改善される）
3. 中野さん（or ミナ）に報告

## 📝 使用上の注意

### 必ず守ること
- ✅ **最終判断は人間 TD が行う**
- ✅ **フィードバックを毎回残す**（学習のため）
- ✅ **状況は具体的に入力する**
- ✅ **API Key は他人に共有しない**

### やってはいけないこと
- ❌ AI の判断をそのまま「これが正解」とプレイヤーに伝える
- ❌ フィードバックをスキップする
- ❌ 曖昧な入力で判断を取る
- ❌ .env ファイルを Git にコミットする

## 🎓 よくある質問

**Q: AI の判断が現場の TD と違ったら？**
A: **必ず現場の TD が正しい**としてください。AI は TDA 条文ベースの推奨なので、現場の文脈（ハウスルール等）を完全には把握できません。差分を ❌ フィードバックで残してください。

**Q: 判断履歴はどこに保存される？**
A: PC のローカル（`data/td_ai.db`）に SQLite で保存されます。バックアップ推奨。

**Q: オフラインで使える？**
A: いいえ。Claude API を呼ぶので**インターネット接続必須**です。

**Q: 他店舗でも使いたい**
A: Phase 4 で**他店舗向けライセンスモード**を実装予定です。現時点では P1 関係者のみ。

**Q: TDA ルールが更新されたら？**
A: 年 1 回程度 TDA が更新するので、その都度ミナがメンテナンスします。
